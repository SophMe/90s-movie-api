Movie API Documentation

Objective

To build the server-side component of a “movies” web application. the web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Context

Creating an REST API for an app that interacts with a database and stores data about different movies.

Features

  • Return a list of ALL movies to the user
  • Return title of a single movie by title to the user
  • Return data about a genre (and description) by name (e.g., “Drama”)
  • Return data about a director (birth year) by name
  • Allow new users to register
  • Allow users to update their user info (username)
  • Allow users to add a movie to their list of favorites (showing only a text that a movie has been added)
  • Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed)
  • Allow existing users to deregister (showing only a text that a user email has been removed

URL Endpoints

Request URL HTTP Method Request body data format Response body data format
Get a list of movies /movies/:title GET None A JSON object holding data about all the movies
Get director by title /movies/director GET - None A JSON object holding data about the director
Get genre by genre /movies/genre/:genre GET :genre None A JSON object holding data about the genre
Get director birthyear by name /movies/director/:name GET :name None A JSON object holding data about a single director
Allow new user to register /users POST - A JSON object holding data about the student to add, structured like: { name : "Charlotte", favoriteMovies: [] } A JSON object holding data about the student to add, including an ID: { id : "1", name : "Charlotte", favoriteMovies: [] }
Allow user to update their username /users/:id UPDATE :id None A JSON object holding data about the updated information
Allow user to add a movie /users/:id/:title POST :id, :title None A JSON Object holding data about the User and the movie that was added
Allow user remove a movie /users/:id/:title DELETE :id, :title None A JSON Object holding data about the User and the movie that was removed
Allow existing user to deregister /users/:id DELETE :id None A text message that the movie was successfully removed
